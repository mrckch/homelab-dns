Quick Actions

Pi-hole neu starten

Container stoppen und wieder hochfahren. ~10 Sek.

cd /opt/homelab-dns/compose && docker compose restart pihole

Alle Container neu starten

Pi-hole + Recovery-Site neustarten.

cd /opt/homelab-dns/compose && docker compose restart

Container neu erzeugen

Bei Konfigurationsaenderungen. Daten bleiben erhalten.

cd /opt/homelab-dns/compose && docker compose up -d --force-recreate

Image-Update prüfen

Neue Pi-hole-Version laden. Versetzt fahren!

cd /opt/homelab-dns/compose && docker compose pull && docker compose up -d

Gravity aktualisieren

Adlists neu einlesen, Blocking-Datenbank bauen.

docker exec pihole pihole -g

Sync jetzt ausführen

Aktion ist rollenabhängig (siehe Befehl).

/opt/homelab-dns/scripts/teleporter-export.sh

System-Status aktualisieren

Erzwingt sofortige Neuberechnung der Dashboard-Werte.

/opt/homelab-dns/scripts/update-site-info.sh

Repo aktualisieren

Neueste Skripte und Konfig holen.

cd /opt/homelab-dns && git pull

Recovery-Szenarien

1. Container kaputt — VM läuft

  • Logs prüfen: docker compose logs --tail=50 pihole
  • Container neustarten: docker compose down && docker compose up -d
  • DNS testen: dig @127.0.0.1 example.com

2. VM kaputt — Host läuft

  • Aus NAS-Snapshot wiederherstellen, ODER:
  • Neue VM aufsetzen, Secrets aus Passwort-Manager hinterlegen
  • Bootstrap holen: curl -fsSL https://raw.githubusercontent.com/mrckch/homelab-dns/main/scripts/bootstrap.sh -o /tmp/bootstrap.sh
  • chmod +x /tmp/bootstrap.sh && sudo /tmp/bootstrap.sh

3. Host komplett tot

  • Andere Pi-hole-Instanz übernimmt automatisch DNS (UDM verteilt beide IPs)
  • Neuer Host + neue VM aufsetzen
  • Bootstrap mit Original-IP — Sync läuft in der nächsten Nacht

Secrets

Secrets niemals auf dieser Seite anzeigen!
  • Pi-hole Passwort: Passwort-Manager → "Homelab / Pi-hole"
  • GitHub PAT (nur Master, /etc/dns-sync/github.token): Passwort-Manager → "Homelab / GitHub PAT"
  • SSH Deploy-Key (nur Follower, /root/.ssh/id_ed25519_homelab-dns): Passwort-Manager → "Homelab / Deploy Key"
