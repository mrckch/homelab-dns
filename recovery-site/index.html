Häufige Befehle

Container-Status prüfen

cd /opt/homelab-dns/compose && docker compose ps

Pi-hole neu starten

cd /opt/homelab-dns/compose && docker compose down && docker compose up -d

Container-Logs ansehen

cd /opt/homelab-dns/compose && docker compose logs -f --tail=100 pihole

DNS-Test

dig @127.0.0.1 example.com

Sync manuell auslösen

/opt/homelab-dns/scripts/teleporter-export.sh

Sync-Log prüfen

tail -30 /var/log/pihole-sync/export.log

Image updaten (nach Runbook!)

cd /opt/homelab-dns/compose && docker compose pull && docker compose up -d

Recovery-Anleitung

  • Container kaputt: docker compose down && docker compose up -d
  • VM kaputt: Aus NAS-Snapshot wiederherstellen oder bootstrap.sh neu ausführen
  • Host tot: Neue VM, Secrets aus Passwort-Manager, bootstrap.sh

Vollständige Anleitung: Recovery-Dokumentation auf GitHub →

Wo finde ich die Secrets?

Secrets werden NIEMALS auf dieser Seite angezeigt.
  • Pi-hole Passwort: Passwort-Manager → "Homelab / Pi-hole"
  • GitHub PAT (nur Master): Passwort-Manager → "Homelab / GitHub PAT"
  • SSH Deploy-Key (nur Follower): Passwort-Manager → "Homelab / Deploy Key"
